# publish-dashboard

## Goal

Publicar automáticamente un historial navegable y verificable.

## Instructions

Cada run debe persistir metadata, fuente, capture-plan, capture-results, imágenes, review y crítica estructurada. Regenerar el dashboard del artefacto y el índice global al cerrar una iteración. Verificar existencia, tamaños y hashes antes de anunciarlo. Mostrar baseline, candidatas, estado de evidencia, comentarios, notas por captura y comparación entre versiones.
