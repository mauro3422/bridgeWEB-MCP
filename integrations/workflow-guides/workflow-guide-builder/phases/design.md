# design

## Goal

Design a guide that activates reliably without taking over unrelated work.

## Instructions

Define the guide's single coherent purpose, scope, activation keywords, exact trigger phrases, positive examples, negative keywords, phases, recommended tools, outputs, and verification rules. Use global scope only for cross-project behavior. Use project scope for repository-specific paths, commands, naming, or quality rules. Require explicit confirmation before destructive actions and before creating the guide when the user has only received a recommendation.
