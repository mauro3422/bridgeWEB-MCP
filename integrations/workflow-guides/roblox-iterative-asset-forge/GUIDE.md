# Roblox Iterative Asset Forge

## Purpose

Turn concept images and written art direction into divergent Roblox ProceduralModel families, stage semantic variants and growth phases, normalize materials, capture comparable evidence, critique against references, iterate, and promote a canonical visual source without repeating weak legacy geometry.

## Activation

Use this guide only when its activation phrases or keywords clearly match the user's task. If the match is uncertain, explain the possible match instead of silently forcing the workflow.

## Workflow

1. **brief-and-contract** — Convert the user request and references into a visual contract with must-preserve features, allowed simplifications, gameplay scale, states, animation/growth needs, and rejection criteria.
2. **divergent-concepts** — Produce genuinely different concept families rather than incremental recolors or deformations.
3. **image-guided-generation** — Generate Roblox ProceduralModel candidates from concept images and named masses.
4. **normalize-and-stage** — Create a persistent isolated review workstation with canonical naming, semantic attributes, anchored display geometry, safe query/collision settings, and consistent lighting/scale.
5. **automatic-review-plan** — Derive deterministic cameras, turnarounds, state strips, animation keyframes, and detail shots from the selected object bounds and metadata.
6. **reference-comparison** — Persist references, renders, overlays, contours, metrics, and a human-readable critique.
7. **iterate-or-reject** — Make one bounded hypothesis per iteration and preserve rejected evidence.
8. **promote-canonical-source** — Promote only a reviewed candidate into a reusable factory or canonical model source.
9. **batch-agent-mode** — Support future autonomous candidate batches while keeping approval human-controlled.
